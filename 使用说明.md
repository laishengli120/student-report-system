# 班主任小助手：期末通知书 AI 生成与送达确认系统使用说明

## 一、案例定位

本系统是面向农村小学期末通知书发放、评语撰写、假期安全告知和家校送达确认的低成本智能信息系统。系统原型和主要代码由班主任借助 DeepSeek 多轮提示词生成、调试和整合，最终形成“AI评语初稿、教师审核、手机通知书送达、家长签收留痕”的闭环。

它不是单一 AI 评语工具，重点解决五个真实问题：班主任评语撰写耗时、纸质通知送达慢、留守儿童父母难以及时知情、假期安全告知分散、回执催收与统计负担重。

## 二、运行环境

- Python 3.10 或以上版本
- 可联网的普通电脑
- DeepSeek 或兼容 OpenAI SDK 的国产大模型接口
- 浏览器：Chrome、Edge、Safari 均可

## 三、安装与启动

```bash
pip install -r requirements.txt
python app.py
```

启动后访问：

- 家长端：`http://127.0.0.1:5000/parent`
- 教师端：`http://127.0.0.1:5000/admin/login`

默认管理员账号：

- 用户名：`admin`
- 密码：`admin123`

正式使用前建议修改默认密码，并妥善保管 AI 接口密钥。

## 四、教师端流程

1. 登录教师端后台。
2. 设置放假、报到、开学等通知书时间。
3. 按学校或班级需要设置分数等级规则，可选择家长端“仅等级”“仅分数”或“分数+等级”显示。
4. 在 AI 接口页面填写 API Key、Base URL 和 Model。
5. 选择寒假版、暑假版或通用版安全告知书模板，也可编辑后调用 AI 优化文案。
6. 上传学生 Excel 表格，可选择“启用 AI 评语审核”。
7. 在评语审核页为学生选择成长画像标签，逐条或批量调用 AI 生成评语草稿。
8. 教师逐条审核、修改、保存，确认无误后统一发布给家长端查看。
9. 发布后在签收看板查看已签收、未签收人数，跟进未签收家庭。

AI 生成内容只作为初稿，必须经过教师审核后才能发布。

## 五、家长端流程

1. 家长在手机浏览器打开家长端页面。
2. 输入学生姓名查询通知书。
3. 查看学生评价、学业等级或分数显示、假期安全告知、放假和开学安排。
4. 在手机端手写签名确认。
5. 系统保存签名图片和签署时间，便于班主任跟进未签收家庭。

对于父母在外务工的留守儿童家庭，家长端可以帮助父母第一时间了解孩子情况，减少纸质通知层层转述造成的信息遗漏。

## 六、AI 配置说明

教师端后台可配置：

- API Key
- Base URL，默认可使用 `https://api.deepseek.com`
- Model，按实际购买或开通的模型填写

系统内置提示词要求 AI 评语只关注身心发展、性格品行、习惯养成和成长建议，不以成绩、分数、考试、学科表现评价学生。

安全告知书的 AI 优化功能同样需要先配置接口。生成后请教师阅读、调整并保存，不建议未经检查直接发布。

## 七、Excel 字段建议

核心字段：

- 姓名
- 语文
- 数学
- 英语

可选字段：

- 科学
- 道德与法治
- 职位
- 期末评语

如果启用 AI 评语审核，系统会把上传记录保存为草稿；如果不启用，则直接发布到家长端。

## 八、参赛与隐私注意

- 演示视频和截图请使用匿名学生数据。
- 不出现学生正面画面、真实联系方式、真实家庭住址和敏感家庭信息。
- 如果材料中出现 AI 生成评语，应标明“AI生成初稿，经教师审核后使用”。
- 录制演示视频时采用“PPT + 系统录屏 + 本人解说”，控制在 8 分钟以内。
- 配套资源 ZIP 建议包含完整代码、README、使用说明、提示词、匿名截图和参赛材料。
- 打包提交前删除以 `~$` 开头的 Word 临时文件。
- 案例信息表中的姓名、单位、手机、签名日期和开源仓库地址需替换为真实信息。

## 九、PythonAnywhere 云端部署

PythonAnywhere 是一个面向 Python 应用的免费/低成本云托管平台，无需购买服务器即可让系统上线运行，非常适合经费有限的乡村学校。免费账号可运行一个 Web 应用，足以支撑期末通知书系统的日常使用。

项目提供了两种部署方式，老师们可根据自身情况选择最方便的一种：

- **方式 A（推荐有 GitHub 账号的老师）**：从 GitHub 仓库直接拉取代码。
- **方式 B（推荐没有 GitHub 账号的老师）**：通过 PythonAnywhere 的 Files 页面上传项目文件。

两种方式共用同一个 `deploy_pythonanywhere.sh` 部署脚本，脚本会自动完成环境配置。

### 9.1 注册 PythonAnywhere 账号

1. 打开浏览器，访问 [pythonanywhere.com](https://www.pythonanywhere.com)。
2. 点击页面右上角的 "Pricing & signup" → 在免费计划 "Beginner" 下方点击 "Create a Beginner account"。
3. 填写用户名、邮箱和密码，完成注册。用户名会决定你的访问地址（如 `https://你的用户名.pythonanywhere.com`），建议用学校或自己的拼音命名。
4. 注册成功后会自动登录，进入仪表盘（Dashboard）。

### 9.2 上传部署脚本到 PythonAnywhere

部署脚本 `deploy_pythonanywhere.sh` 在项目根目录中。需要先把脚本传上 PythonAnywhere，有两种方法，任选其一即可。

**方法一：通过 Files 页面上传（推荐）**

1. 在 PythonAnywhere 顶部导航栏，点击 **"Files"** 标签。
2. 页面左侧是目录树，确认当前在 `/home/你的用户名/` 目录下（默认就是）。
3. 点击中间的 **"Upload a file"** 按钮。
4. 在弹出的文件选择框中，从本地电脑找到 `deploy_pythonanywhere.sh` 文件，选中后点击"打开"。
5. 等待上传完成，页面刷新后你应该能在文件列表中看到 `deploy_pythonanywhere.sh`。

**方法二：直接在 Bash 中创建文件**

1. 在 PythonAnywhere 顶部导航栏，点击 **"Consoles"** → 点击 **"Bash"**，打开一个黑色的命令行终端。
2. 在本地电脑上，用记事本或 VS Code 打开 `deploy_pythonanywhere.sh`，全选复制全部内容。
3. 回到 PythonAnywhere 的 Bash 终端，输入以下命令后回车：

```bash
cat > deploy_pythonanywhere.sh
```

4. 此时光标会停在新的一行等待输入。把刚才复制的脚本内容**粘贴**进去（在终端中右键 → Paste，或按 Ctrl+Shift+V）。
5. 粘贴完成后，按一下回车换行，然后按 **Ctrl+D** 结束输入。文件就创建好了。
6. 输入 `ls` 回车，确认能看到 `deploy_pythonanywhere.sh`。

### 9.3 运行部署脚本

1. 在 PythonAnywhere 顶部导航栏，点击 **"Consoles"** → **"Bash"**，打开 Bash 终端（如果之前关掉了的话）。
2. 确认脚本文件在当前位置：

```bash
ls
# 应该能看到 deploy_pythonanywhere.sh
```

3. 给脚本添加执行权限并运行：

```bash
chmod +x deploy_pythonanywhere.sh
bash deploy_pythonanywhere.sh
```

4. 脚本启动后，会先让你确认用户名（直接回车即可），然后询问选择哪种代码获取方式。

### 9.4 方式 A：从 GitHub 拉取（有 GitHub 账号）

如果已经将项目代码上传到了 GitHub 仓库，这是最省事的方式，后续更新也方便。

1. 当脚本询问 "请选择项目代码的获取方式" 时，输入 `1` 并回车。
2. 输入你的 Git 仓库地址（如 `https://github.com/用户名/仓库名.git`），回车。
3. 脚本会自动克隆代码、创建虚拟环境、安装依赖、配置 WSGI 文件。
4. 脚本执行完毕后，按照屏幕提示完成后续手动配置（详见 9.6 节）。

后续每次更新代码只需在 Bash 中运行：

```bash
cd ~/student-report-system && git pull
```

然后点击 Web 页面顶部的绿色 **"Reload"** 按钮即可。

### 9.5 方式 B：从本地上传项目（无 GitHub 账号）

如果没有 GitHub 账号，可以直接把项目文件打包上传到 PythonAnywhere。

**第一步：在本地电脑上准备项目压缩包**

1. 打开项目文件夹（就是包含 `app.py`、`ai_service.py`、`templates/`、`static/` 等文件的文件夹）。
2. 全选所有 `.py`、`.html`、`.css`、`.js`、`.txt`、`.md` 文件，以及 `templates/` 和 `static/` 文件夹。
3. **注意：一定不要选中 `.venv` 文件夹、`__pycache__` 文件夹、`instance/` 和 `uploads/` 文件夹**（这些在服务器上会重新生成）。
4. 把选中的文件和文件夹压缩为一个 `.zip` 文件：
   - **Windows**：右键选中的文件 → "发送到" → "压缩(zipped)文件夹"，命名为 `report-system.zip`。
   - **Mac**：右键选中的文件 → "压缩"，重命名为 `report-system.zip`。

**第二步：上传压缩包到 PythonAnywhere**

1. 在 PythonAnywhere 顶部导航栏，点击 **"Files"** 标签。
2. 确认当前在 `/home/你的用户名/` 目录下。
3. 点击 **"Upload a file"** 按钮，选择刚才压缩好的 `report-system.zip`。
4. 等待上传完成（文件较大可能需要几十秒，上传进度在页面顶部显示）。

**第三步：解压项目文件**

1. 上传完成后，点击顶部导航的 **"Consoles"** → **"Bash"**，打开或回到 Bash 终端。
2. 执行解压命令：

```bash
# 先确认 zip 文件已经上传成功
ls
# 应该能看到 deploy_pythonanywhere.sh 和 report-system.zip

# 解压到 student-report-system 文件夹
unzip report-system.zip -d student-report-system
```

3. 解压完成后，可以检查一下关键文件是否存在：

```bash
ls student-report-system/
# 应该能看到 app.py、ai_service.py、requirements.txt、wsgi.py 等文件
```

**第四步：运行部署脚本**

回到 9.3 节的步骤运行部署脚本（如果还没运行的话）：

1. 当脚本询问 "请选择项目代码的获取方式" 时，输入 `2` 并回车。
2. 输入解压后的项目文件夹路径，即 `/home/你的用户名/student-report-system`，回车。
3. 脚本会自动创建虚拟环境、安装依赖、配置 WSGI 文件。
4. 脚本执行完毕后，按照屏幕提示完成后续手动配置（详见 9.6 节）。

**第五步：后续更新代码**

当项目有新版本时，重新打包文件上传覆盖原来的文件夹（在 Files 页面删除旧的 `student-report-system` 文件夹，再按 9.5 节的步骤重新上传解压），然后点击 Web 页面的绿色 **"Reload"** 按钮即可。

### 9.6 手动配置 Web 应用

无论使用哪种方式，脚本运行完毕后，还需要在 PythonAnywhere 网页界面上完成以下配置。

**a) 创建 Web 应用**

1. 进入 PythonAnywhere 仪表盘，点击顶部导航的 **"Web"** 标签。
2. 点击 **"Add a new web app"** 按钮。
3. 在弹出的向导中，点击 "Next" 跳过域名选择，然后选择 **"Manual configuration"**（注意：不要选 "Flask"，我们要手动配置）。
4. Python 版本选择 **3.10** 或更高版本，点击 "Next" 完成创建。

**b) 配置 WSGI 文件**

创建完成后，页面会显示 Web 应用的配置信息。在 **"Code"** 区域有一个 WSGI 配置文件链接，点击进入编辑器。

1. 回到 Bash 终端，执行以下命令查看脚本为你生成的 WSGI 内容：

```bash
cat ~/wsgi_config.txt
```

2. 全选复制终端输出的所有内容。
3. 回到 WSGI 编辑器中，**清空原有全部内容**，把刚复制的内容粘贴进去。
4. 点击编辑器右上角的 **"Save"** 按钮保存。
5. 点击顶部导航的 "Web" 回到 Web 配置页面。

**c) 关联虚拟环境**

在 Web 配置页面的 **"Virtualenv"** 栏，填入虚拟环境路径：

```
/home/<你的用户名>/.virtualenvs/student-report-env
```

填写后点击旁边的确认按钮。

**d) 配置静态文件映射**

在 **"Static files"** 区域，填写并添加一条映射：

| URL      | Directory                                        |
|----------|--------------------------------------------------|
| /static/ | /home/<你的用户名>/student-report-system/static/ |

**e) 重新加载应用**

点击页面顶部的 **绿色 "Reload" 按钮**，等待几秒后配置即生效。

### 9.7 访问系统

部署完成后即可通过以下地址访问：

- 家长端：`https://<你的用户名>.pythonanywhere.com/parent`
- 教师端：`https://<你的用户名>.pythonanywhere.com/admin/login`

默认管理员账号：`admin` / `admin123`（首次登录后请立即修改密码）。

### 9.8 部署后必做事项

1. **修改默认密码**：登录教师端后，建议通过修改代码中的 `DEFAULT_ADMIN_PASSWORD` 或在数据库中更新密码哈希值。
2. **配置 AI 接口**：在教师端后台 "AI 接口" 页面填写 DeepSeek 或其他兼容接口的 API Key、Base URL 和 Model。
3. **设置 HTTPS**：PythonAnywhere 默认启用 HTTPS，免费域名即可安全访问。

### 9.9 GitHub Webhook 自动更新（仅方式 A）

系统内置了 GitHub Webhook 支持（`/git-update` 路由），可在 GitHub 仓库 Settings → Webhooks 中配置 Payload URL：

```
https://<你的用户名>.pythonanywhere.com/git-update
```

配置后每次推送代码到 GitHub，PythonAnywhere 会自动拉取最新代码并重新加载应用。

### 9.10 免费账号限制说明

PythonAnywhere 免费账号有以下限制，对学校使用场景一般够用：

- 只能运行 1 个 Web 应用
- 每天有 CPU 使用时间限制（约 100 秒高强度运算）
- 不能使用自定义域名（使用 `*.pythonanywhere.com` 子域名）
- 数据库使用项目自带的 SQLite，无需额外配置
- 如需更大资源或自定义域名，可升级到 Hacker 计划（$5/月）

如果系统需要处理大量并发查询，建议升级到付费计划或迁移到其他云平台。
